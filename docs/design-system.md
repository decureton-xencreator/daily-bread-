# XDBS 2.0 Design System

The system uses semantic tokens in `assets/css/xdbs.css` for surface, ink, muted text, borders, cyan orientation, violet learning, gold caution, green completion and red risk. Components include the application shell, fixed rail, mobile navigation, arrival hero, priority rows, course cards, intelligence cards, source/confidence/freshness badges, progress bars, globe, media cards, mission panel, radar cards, timeline, archive cards, dialogs, drawers, toast, unavailable states and diagnostic rows.

Breakpoints: wide desktop by default, adaptive tablet/laptop below 1000px, and purpose-built mobile below 700px. Reduced-motion and increased-contrast media queries are included. Touch controls meet a 40–48px target wherever practical.
