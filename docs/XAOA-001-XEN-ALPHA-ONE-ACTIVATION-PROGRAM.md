# XAOA-001 — Xen Alpha One Activation Program

Status: **Canonical / Active**  
Priority: **Priority Zero**  
Runtime gate: `app/xaoa-program.js`

## Mission

Transform the completed Xen Alpha One reference environment into a persistent, daily-use, phone-first operating product running on live infrastructure, validated by Warden, and proven through the first Living Company.

## Ordered activation contract

The executable controller evaluates XRI-006 through XRI-010 individually, then identity, storage, memory, model and event bindings, the Daily Bread → Mission Control → Command → Memory → Warden loop, phone certification, measured Living Company value, production/recovery/rollback drills, and XBP-009 Gold Master certification.

Every gate requires a positive result, an evidence identifier and a timestamp. A failed or missing predecessor blocks every successor. Phone certification additionally requires installability, offline recovery and touch-target review. The Living Company gate requires a named pilot and at least one measured baseline/result pair. XBP-009 is the only terminal Gold Master authority.

## Truth boundary

This repository implements and tests the activation controller. It does not convert absent provider connections, unmeasured pilot outcomes, unperformed drills, or an unrun XBP-009 assessment into completion evidence. Until those receipts exist, the program remains active at the earliest unproven gate.
