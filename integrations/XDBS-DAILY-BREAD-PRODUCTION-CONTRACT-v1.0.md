# XDBS Daily Bread Production Contract

**Version:** 1.0  
**Status:** Active  
**Canonical architecture:** `decureton-xencreator/xen-operating-system/architecture/xdbs/XDBS-001-XEN-DAILY-BREAD-STUDIO-ARCHITECTURE-v1.0.md`

## Production rule

This repository implements Daily Bread only through XDBS and XPS standards.

Daily Bread SHALL NOT be published as markdown-only, plain-text, chat-prose, or legacy report output. A failed HTML build must fail visibly rather than silently downgrade.

## Required edition modules

- verified date, time, location, and operating mode;
- Academy Command;
- World in 5 Minutes with tappable sources;
- weather and travel context;
- markets when relevant;
- risk and opportunity radar;
- Water Cooler bullet cards;
- bespoke Sports, Entertainment, and XMI modules;
- Xen Build Evolution;
- Xen Globe integration;
- focused execution plan;
- provenance and verification note.

## Repository publishing target

```text
/
  index.html
  archive/YYYY-MM-DD.html
  assets/
  components/
  templates/
  data/
  releases/
```

`index.html` represents the current released edition. Every released edition must also be preserved in the dated archive.

## Validation gates

- responsive mobile layout;
- functional navigation and external links;
- accessible typography, contrast, and touch targets;
- factual source links for current affairs;
- truthful runtime claims;
- archive creation;
- release metadata;
- no private data exposed in public output without permission.

## Capability integrations

- XGS — Xen Globe Studio
- XAS — Xen Academy Studio
- XMI — Xen Music Intelligence
- Sports and Entertainment intelligence
- Travel, Weather, Markets, Calendar, Memory, Presence, XCR, and XP-002

Architecture does not equal implementation. Features may be displayed as preview concepts only when clearly labeled.
